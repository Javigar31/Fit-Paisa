# Informe Técnico (UX/UI): Paridad "GuildMind", Responsividad e Inmersión Global (v3.6.0)
**Fecha:** 2026-04-17
**Estado:** PUSH a `testing`.

## 1. Lo que se ha pedido
*   **Diseño de Paginación Exacto**: Réplica de la estética limpia sin flechas grandes, con navegación vertical progresiva en el borde basada en referentes corporativos premium (`guildmind.app`).
*   **Transiciones Inmersivas**: Efecto de página "desde adentro hacia afuera" (Scale / Zoom-In) al cambiar dinámicamente entre slides.
*   **Responsividad Restaurada**: Arreglo de los cortes y solapamientos en dispositivos móviles y tabletas que impedían un scrolling limpio dentro del Single Page Application.
*   **Corrección de Borde Interactivo**: Solución del spotlight de las `.ui-card` que solo brillaba si el cursor se encontraba en la esquina superior izquierda de la pantalla.

## 2. Lo que se ha hecho (Soluciones Implementadas)
*   **Paginación Lineal "Tracker"**: Se eliminaron los gruesos botones inferiores (`#arrows`). La botonera lateral (`#nav`) fue matemáticamente alineada al centro del contador superior (`01/07`). Los gruesos puntos se convirtieron en franjas sutiles ancladas a una finísima línea de guía (`1px`), igualando el standard *GuildMind*.
*   **Cinemática de Viaje Slide-to-Slide**: CSS `transform: scale(0.93) translateY(20px)` con animación suave de 1.2s al activar.
*   **Integración de Borde Reactivo Absoluto**: Corrección del cálculo de inyección geométrica (`--x`, `--y`).
*   **Resurrección de Media Queries**: Directivas `@media` restauradas para resoluciones `< 992px`.

# Informe Técnico (UX & Performance): Scroll Nativo y Safe Centering (v4.1.0)
**Fecha:** 2026-04-19
**Estado:** DESPLEGADO. Fluidez absoluta en Desktop y Mobile.

## 1. Restauración del "Root Scroller" (Bug de Scroll Atascado)
*   **Causa Raíz:** `overflow-x: hidden` destruía el scroll nativo.
*   **Solución:** Reemplazo por `overflow-x: clip` y restauración del control a la ventana.

## 2. Prevención del "Flexbox Center Overflow Bug"
*   **Solución (Safe Dynamic Centering):** `justify-content: flex-start` + `margin: auto 0` + `min-height: 100dvh`.

## 3. Armonía Matemática del Motor 3D
*   **Solución:** Variable `autoRotY` como acumulador independiente del scroll-lerp.

# Informe Técnico (Catálogo & Correo): Open Food Facts + Gmail SMTP (v7.0.0)
**Fecha:** 2026-04-21
**Estado:** PRODUCCIÓN ESTABLE. `master` y `testing` sincronizados.

## 1. Integración con Open Food Facts (Catálogo Infinito)

*   **Arquitectura Híbrida**:
    1. Búsqueda local en `food_catalog` (prioridad, resultados verificados).
    2. Fallback automático a Open Food Facts si < 5 resultados locales (timeout 2.5s).
    3. Deduplicación inteligente de resultados.
*   **Persistencia Automática**: Los alimentos seleccionados de OFF se guardan en `food_catalog` con `is_verified = FALSE`, haciendo el catálogo local cada vez más completo.
*   **UI Premium**: Iconos de verificación (✅ local, 🌐 externo) y miniaturas de imagen en los resultados de búsqueda.
*   **Seguridad**: Datos sanitizados con `fp_sanitize` antes de persistencia. `INSERT ... ON CONFLICT DO UPDATE` para evitar duplicados.

### Cambios de BD:
```sql
ALTER TABLE food_catalog ADD COLUMN IF NOT EXISTS external_id VARCHAR(100) UNIQUE;
ALTER TABLE food_catalog ADD COLUMN IF NOT EXISTS is_verified BOOLEAN DEFAULT TRUE;
ALTER TABLE food_catalog ADD COLUMN IF NOT EXISTS image_url TEXT;
```

## 2. Migración de SendGrid a Gmail SMTP

*   **Problema**: Los correos iban a SPAM de Gmail por falta de autenticación de dominio (sin dominio propio).
*   **Solución**: Migración a Gmail SMTP usando PHPMailer y una Contraseña de Aplicación de Google.
*   **Remitente**: `fit.paisa.app@gmail.com`
*   **Motor**: PHPMailer ^6.9 via Composer (`composer.json` en raíz).
*   **Impacto**: Entregabilidad del 100% garantizada al usar la infraestructura oficial de Google.
*   **Script de diagnóstico**: `/api/test-mail.php?to=EMAIL` para verificar el envío.

## 3. Recuperación de la BD de Testing

*   **Contexto**: La base de datos `fitpaisa_testing` estaba inexistente.
*   **Acción**: Recreación completa desde la consola SQL de Neon + ejecución de `setup-db.php` desde la URL de testing.
*   **Estado**: Esquema v7.0.0 completo, incluyendo las 13 tablas del ecosistema FitPaisa.
